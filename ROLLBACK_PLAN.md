# План отката к «вчерашнему» состоянию с сохранением английской сборки

**Важно:** В Git у вас нет коммита за 7 февраля. Последние коммиты:
- `7c5c691` — 2026-02-03 (Remove build from tracking)
- `85660c0` — 2026-02-03 (Checkpoint 03.02.2-Elements)

Поэтому «откат к вчерашнему вечеру» возможен двумя способами:
- **Вариант А** — откатить код по Git к состоянию **3 февраля** (последний зафиксированный «хороший» коммит), затем вручную вернуть только изменения, нужные для английской сборки.
- **Вариант Б** — если у вас включена «История файлов» или «Предыдущие версии» Windows, восстановить папку проекта на дату 7 февраля и при необходимости снова добавить английскую сборку.

Ниже — пошаговый план с чекбоксами. Отмечайте шаги по мере выполнения.

## Подготовка (обязательно перед любым откатом)

- [ ] **1.1** Закрыть приложение на телефоне/эмуляторе и все терминалы, где запущен `flutter run` или сборка этого проекта.
- [ ] **1.2** Создать полную резервную копию текущей папки проекта (например, скопировать `Midnight_Dancer backup 31.01.26` в `Midnight_Dancer_backup_08-02-2026`). Так вы сможете вернуться к текущему состоянию, если что-то пойдёт не так.
- [ ] **1.3** Сохранить копии файлов, которые нужны для английской сборки (см. раздел «Файлы английской сборки» ниже), в отдельную папку, например `C:\MD_english_build\`:
  - [ ] `lib/core/app_flavor.dart`
  - [ ] `lib/core/app_strings.dart` (целиком)
  - [ ] `lib/app.dart`
  - [ ] `android/app/build.gradle`
  - [ ] `lib/ui/screens/choreography/sequence_editor_screen.dart` (если он у вас есть и без него английская сборка не запускалась)

## Вариант А: Откат по Git к 3 февраля + возврат английской сборки

Выполнять **строго по шагам**, отмечая чекбоксы.

- [ ] **А.1** Открыть PowerShell (или терминал) и перейти в папку проекта:
  ```powershell
  cd "c:\Applications\Midnight_Dancer backup 31.01.26\Midnight_Dancer"
  ```
- [ ] **А.2** Убедиться, что вы в репозитории и видите последние коммиты:
  ```powershell
  git log --oneline -5
  ```
  Должны быть коммиты `7c5c691` и `85660c0` от 2026-02-03.
- [ ] **А.3** Откатить все отслеживаемые файлы к коммиту от 3 февраля (последний «нормальный» коммит):
  ```powershell
  git reset --hard 7c5c691
  ```
  **Внимание:** неотслеживаемые файлы (например, `app_strings.dart`, часть скриптов) при этом **не удаляются**. Изменения в отслеживаемых файлах будут потеряны.
- [ ] **А.4** Очистить сборку Flutter и кэш:
  ```powershell
  flutter clean
  flutter pub get
  ```
- [ ] **А.5** Вернуть файлы английской сборки из папки, куда вы их сохранили в п. 1.3:
  - Скопировать обратно в проект: `app_flavor.dart`, `app_strings.dart`, `app.dart`, `build.gradle`, при необходимости `sequence_editor_screen.dart`.
- [ ] **А.6** Проверить, что в `pubspec.yaml` есть всё необходимое для запуска (зависимости не удалены). При необходимости сравнить с резервной копией из п. 1.2.
- [ ] **А.7** Запустить русскую сборку (lite) и убедиться, что приложение открывается и основные экраны работают:
  ```powershell
  flutter run --flavor lite --dart-define=FLAVOR=lite
  ```
- [ ] **А.8** Запустить английскую сборку и убедиться, что она тоже работает:
  ```powershell
  flutter run --flavor english --dart-define=FLAVOR=english
  ```

Если после **А.3** что-то пошло не так, можно вернуть состояние до отката (если не делали новых коммитов):
```powershell
git reflog
git reset --hard HEAD@{0}
```
(подставьте нужный номер из `reflog` для состояния «до» reset).

- [ ] **Б.1** Закрыть Cursor/IDE и все процессы, использующие файлы проекта.
- [ ] **Б.2** В проводнике перейти в родительскую папку проекта (например, `c:\Applications\` или ту, где лежит `Midnight_Dancer backup 31.01.26`).
- [ ] **Б.3** Щёлкнуть правой кнопкой по папке `Midnight_Dancer backup 31.01.26` → **Свойства** → вкладка **Предыдущие версии** (или «Восстановить прежнюю версию»).
- [ ] **Б.4** Если в списке есть версия с датой **7 февраля 2026** (вечер), выбрать её и нажать **Восстановить** (или «Восстановить в…» в отдельную папку, чтобы не перезаписать текущую).
- [ ] **Б.5** Если восстановили в отдельную папку — скопировать из неё в рабочую папку проекта только нужные файлы (или заменить папку целиком, предварительно сделав копию текущей по п. 1.2).
- [ ] **Б.6** Проверить, что английская сборка на месте (если в той версии её ещё не было — подставить файлы из п. 1.3 и снова проверить запуск).

## Файлы английской сборки (что хранить в п. 1.3)

Чтобы после отката не потерять английскую сборку, сохраните копии этих файлов:

| Файл | Зачем нужен |
|------|-------------|
| `lib/core/app_flavor.dart` | Определение `isEnglish` и flavor `english`. |
| `lib/core/app_strings.dart` | Все строки интерфейса RU/EN (файл мог быть не в Git). |
| `lib/app.dart` | `locale: AppFlavor.isEnglish ? const Locale('en') : null`, `supportedLocales`. |
| `android/app/build.gradle` | Секция `english { applicationIdSuffix ".english" }` в `productFlavors`. |
| `lib/ui/screens/choreography/sequence_editor_screen.dart` | Исправление для запуска английской сборки (если без него был краш/ошибка). |

Конфигурация запуска (`.vscode/launch.json`, `.idea/...`) лежит выше папки `Midnight_Dancer` и при откате внутри `Midnight_Dancer` не изменится.
