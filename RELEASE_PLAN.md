## Шаг 0. Подготовка (один раз перед первым релизом)

### 0.1 Релизная подпись Android

Сейчас в проекте для release используется debug-подпись. Для магазинов нужна своя подпись.

1. Создать keystore (в папке проекта или в безопасном месте):
   ```
   keytool -genkey -v -keystore upload-keystore.jks -keyalg RSA -keysize 2048 -validity 10000
   ```
   Указать пароль, ФИО, организацию и т.д. Файл `upload-keystore.jks` и пароли хранить в безопасности и не коммитить в git.

2. В папке `android/` создать файл `key.properties` (не добавлять в git):
   ```
   storePassword=ВАШ_ПАРОЛЬ
   keyPassword=ВАШ_ПАРОЛЬ
   keyAlias=upload
   storeFile=../upload-keystore.jks
   ```
   (путь к storeFile — относительно папки android или абсолютный)

3. В `android/app/build.gradle` в блоке `android { ... }`:
   - прочитать `key.properties` в начале файла (через `def keystoreProperties = new Properties()` и `keystoreProperties.load(...)`);
   - добавить `signingConfigs { release { ... } }` с полями из keystoreProperties;
   - в `buildTypes.release` указать `signingConfig signingConfigs.release` вместо `signingConfigs.debug`.

## Шаг 1. Сборка Android-билдов (все три варианта)

Скрипт собирает сразу три варианта приложения.

1. В корне проекта (Windows) выполнить:
   ```
   .\scripts\build_apks.ps1
   ```
   В `android\app\build\outputs\flutter-apk\` появятся три APK:
   - **Lite (1 голос):** `app-lite-release.apk`
   - **Standard (2 голоса):** `app-standard-release.apk`
   - **Full (3 голоса):** `app-full-release.apk`

2. Для RuStore и Huawei подойдут APK или AAB. Для Google Play удобнее AAB. Собрать AAB для нужных вариантов (для всех трёх или только для тех, что пойдут в Play):
   ```
   flutter build appbundle --flavor lite --release --dart-define=FLAVOR=lite
   flutter build appbundle --flavor standard --release --dart-define=FLAVOR=standard
   flutter build appbundle --flavor full --release --dart-define=FLAVOR=full
   ```
   Файлы: `build\app\outputs\bundle\release\app-lite-release.aab`, `app-standard-release.aab`, `app-full-release.aab`.

В каждый магазин загружаете те варианты, которые решили публиковать (Lite, Standard, Full — один, два или все три).

- [ ] Готовы к загрузке файлы выбранных вариантов (APK и/или AAB)
